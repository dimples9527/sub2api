# 仓库协作规则

## 前端组件使用

- 前端页面改动只能使用当前项目已有组件；不得为满足页面需求而修改、优化或扩展现有通用组件的内容。
- 如果现有组件能力不足，应在页面层通过组合、适配或显式数据转换解决；确需调整组件本身时，必须先征得用户明确同意。

## 提交信息

- 提交信息必须使用类型前缀，例如 `feat:`、`fix:`、`style:`、`refactor:`、`test:`、`docs:`、`chore:`、`build:`。
- 冒号后的描述必须使用中文。
- 描述要简洁、具体，只概括本次提交的变更。

示例：

- `feat: 新增上游账号删除操作`
- `fix: 修复上游同步日志刷新后丢失`
- `style: 固定上游账号表格列宽`
