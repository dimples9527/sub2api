# 后续开发功能清单

本文记录上游管理相关功能的完成状态和后续增强方向。已完成的功能保留验收口径，便于后续回归；未完成项按优先级继续拆分。

## 当前完成状态

| 优先级 | 功能 | 状态 | 备注 |
| --- | --- | --- | --- |
| P0 | 上游 token 缓存 | 已完成 | 已支持按 provider 复用登录态，认证失效后刷新并重试。 |
| P1 | 上游分组页面展示优化 | 已完成 | 页面改为更直观的只读匹配结果，并保留未匹配同步入口。 |
| P1 | 上游账号绑定分组和倍率展示 | 已完成 | 已匹配账号可展示全部绑定分组、倍率和低倍率风险。 |
| P2 | 账号倍率守护定时解绑 | 已完成 | 已支持默认关闭、间隔配置、进程内防重入、运行状态记录。 |
| P2 | 账号倍率守护手动执行和轮询日志 | 已完成 | 已支持立即执行；页面展示当前进程最新 10 条轮询日志。 |

## 已完成能力验收口径

### 1. 上游 token 缓存

验收标准：

- 同一个 provider 连续拉取数据时，不应每次重新登录。
- token 失效后会自动重新登录并重试原请求。
- 并发请求不会同时触发大量重复登录。
- 日志和接口响应不得泄露 token、cookie、密码或完整 key。

### 2. 上游分组页面优化

验收标准：

- 普通查看场景不需要通过下拉理解匹配关系。
- 未匹配上游分组可以同步为本地分组。
- 倍率风险能明确提示上游倍率与本地倍率差异。

### 3. 上游账号页面优化

验收标准：

- 已匹配账号能看到全部绑定分组和倍率。
- 低于上游倍率的分组能被标出，并与同步后解绑行为一致。
- 创建账号场景仍显示将绑定的目标分组。

### 4. 账号倍率守护

验收标准：

- 配置关闭时不会执行解绑。
- 配置开启且到达间隔后会自动检查并解绑低倍率分组。
- 失败时写入最近失败状态，不影响下一轮执行。
- 手动执行和定时执行共用同一把进程内锁，避免并发解绑。
- 页面能看到最近运行状态和最新 10 条轮询日志。

## 下一步候选功能

### A. 倍率守护审计增强

目标：

- 将“轮询日志”和“解绑明细”区分展示。
- 轮询日志继续保留最新 10 条内存状态，用来确认调度是否活跃。
- 解绑明细使用已有同步记录展示，并补充触发来源：手动同步、定时守护、手动守护。

验收标准：

- 管理页能区分“本次有没有轮询”和“本次实际解绑了什么”。
- 审计记录能看出解绑来自哪种触发方式。
- 定时守护没有解绑时，也能在轮询日志中看到跳过或成功检查记录。

### B. 上游账号匹配冲突处理

目标：

- 当同一个规范化名称匹配多个本地账号时，提供明确处理入口。
- 支持查看冲突账号列表，并允许管理员选择一个账号作为绑定目标或跳过。

验收标准：

- 冲突行不自动创建或更新账号。
- 页面能展示冲突账号 ID、名称和当前绑定分组。
- 管理员处理后，下一次预览不再显示同一冲突。

### C. 上游管理回归测试补齐

目标：

- 为上游 provider token 缓存、分组匹配、账号同步、倍率守护补齐关键回归测试。
- 优先覆盖过去已经实现但容易回归的规则。

验收标准：

- token 复用和失效刷新有单元测试。
- 上游分组未匹配同步有 handler 或 service 测试。
- 账号倍率守护的手动执行、定时执行、防重入、日志保留都有测试。

## 推荐实施顺序

1. 做“倍率守护审计增强”，把触发来源写入记录，方便排查自动解绑来源。
2. 做“上游账号匹配冲突处理”，补齐当前只提示冲突但不能处理的闭环。
3. 做“上游管理回归测试补齐”，降低后续改动风险。
